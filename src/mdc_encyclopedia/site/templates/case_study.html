Using AI to Make Public-Sector Data Understandable and Actionable

I build AI-assisted civic data and workflow systems that turn dense public-sector information into searchable, understandable, operational tools.

The goal is not to use AI as decoration. The goal is to use AI where it fits: summarizing complex source material, enriching metadata, improving search and discovery, accelerating analysis, and helping non-technical users understand what data means and how to act on it.

The Problem

Public-sector information is often technically public but practically hard to use. Budgets, open-data catalogs, department documents, infrastructure records, and service workflows are spread across PDFs, ArcGIS portals, spreadsheets, internal systems, and agency-specific terminology.

  • Residents cannot easily understand what government is funding or doing.
  • Staff spend time translating the same information repeatedly.
  • Departments maintain data that is technically available but poorly explained.
  • Decision-makers lack clean, plain-English views of complex systems.
  • Civic transparency depends on tools that most users will never touch.

Artifact 1: BudgetExplorer

BudgetExplorer turns a dense public budget into an interactive, searchable, plain-English product with an interactive budget treemap, department drilldowns, full-text search, AI-generated department summaries, glossary support, a property-tax allocation calculator, PostgreSQL-backed data modeling, a Next.js / TypeScript frontend, D3/Recharts visualizations, and a Python PDF extraction and validation pipeline.

AI Role

AI helps convert complex budget source material into plain-English summaries that residents and non-specialists can understand. The system still depends on structured extraction, validation, data modeling, and product design; AI is one layer in the workflow, not a substitute for the workflow.

Human Validation

The budget data pipeline includes extraction and validation steps before publication. The point is to make AI outputs useful inside a controlled public-information workflow rather than publishing ungrounded summaries.

Artifact 2: DataDictionary / MDC Encyclopedia

DataDictionary ingests ArcGIS open-data metadata from Miami-Dade, Broward, and City of Miami hubs, then publishes a searchable static encyclopedia with dataset descriptions, field-level explanations, civic use cases, keywords, relevance signals, quality audits, freshness/completeness/documentation grades, SQLite-backed processing, static HTML output, client-side search, and weekly automated refresh through GitHub Actions.

AI Role

AI enriches sparse or technical dataset metadata with structured explanations, use cases, keywords, and civic relevance. This makes public datasets easier to discover, evaluate, and apply.

Human Validation

The project includes quality scoring and structured metadata checks so the catalog does not merely summarize datasets; it also evaluates whether the underlying data is fresh, complete, and documented well enough to be useful.

The Implementation Pattern

  1. Start with messy public-sector source material.
  2. Extract or ingest the data into a structured pipeline.
  3. Validate the data before presenting it.
  4. Use AI to improve explanation, discovery, and usability.
  5. Publish a searchable product that non-technical users can actually use.
  6. Keep the output tied to source data, not generic AI-generated text.

Public-Sector Value

These projects show how AI can improve government operations without pretending that every problem requires a custom model. The practical value is in implementation: making public information easier to understand, reducing repeated staff translation work, improving civic data discovery, helping residents and stakeholders navigate complex systems, turning fragmented data into usable internal or public-facing tools, and creating a foundation for better reporting, triage, knowledge management, and decision support.
