**pyproject.toml**: Add new dependencies: `Jinja2>=3.1`, `lunr>=0.8`, `python-slugify>=8.0`.

**site/__init__.py**: Empty init file.

**site/context.py**: Query helper module that reads from SQLite and builds template-ready dicts. Functions:
- `build_site_data(conn) -> dict` — master function returning all data needed for all pages. Returns dict with keys: datasets (list of dicts), categories (dict mapping category->dataset list), stats (total_datasets, pct_described, new_this_week), changes (list of change dicts), quality_summary (grade distribution, findings), generated_at (ISO timestamp string).
- `_build_dataset_context(row) -> dict` — converts a DB row (from JOIN across datasets, enrichments, audit_scores) into a template-ready dict. Parse JSON fields (tags, keywords, use_cases, findings_json) into Python lists. Compute relative_time from updated_at. Compute staleness_color from audit freshness_score. Compute letter grade CSS class. Generate slug via python-slugify from title (with dataset ID suffix for collision handling).
- `_build_related_datasets(dataset, all_datasets) -> list` — returns top 5 related datasets by: same category (3 points) + each shared keyword (1 point). Return list of {title, slug, department, grade} dicts.
- `_relative_time(dt_str) -> str` — "Updated 3 days ago" style strings. Handle None -> "Unknown".
- `_staleness_color(freshness_score) -> str` — maps score to "green"/"yellow"/"red"/"gray".
- `_grade_class(letter_grade) -> str` — maps grade to CSS class (e.g., "grade-a").

The master query in build_site_data should do a LEFT JOIN across datasets, enrichments, and audit_scores to get all data in one pass. Use the existing get_connection pattern from db.py. Also query the columns table and the changes table.

**site/generator.py**: Main export orchestrator. Functions:
- `generate_site(db_path: str, output_dir: str = "site") -> dict` — reads DB via context.py, sets up Jinja2 Environment with FileSystemLoader pointing to templates/ dir (using `os.path.dirname(__file__)` for path resolution), enables autoescape, trim_blocks, lstrip_blocks. Registers custom Jinja2 filters: `relative_time`, `staleness_color`, `grade_class`. Creates output directory structure (site/, site/browse/, site/dataset/, site/changes/, site/quality/, site/about/, site/static/). Calls individual render functions for each page type. Copies static assets. Returns stats dict with page counts and output path.
- `_render_page(env, template_name, context, output_path)` — renders a single template to a file. Creates parent dirs as needed.
- `_copy_static_assets(output_dir)` — copies style.css and search.js from site/static/ source to output site/static/.
- Individual render functions: `_render_homepage`, `_render_browse_pages`, `_render_dataset_pages`, `_render_changes_page`, `_render_quality_page`, `_render_about_page`. Each takes env, site_data, output_dir. For now, all should work with the base template, rendering minimal content (just a heading). The full template content comes in Plans 02-04.

For slug collision detection: maintain a seen_slugs dict during dataset context building. If slug already exists, append `-{dataset_id[:8]}` to disambiguate. Log a warning.
