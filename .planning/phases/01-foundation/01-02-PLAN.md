  <files>src/mdc_encyclopedia/cli.py</files>
  <action>
Create `src/mdc_encyclopedia/cli.py` implementing:

1. Use `import rich_click as click` (drop-in replacement pattern per research -- NOT `import click` separately). Also import `from rich.console import Console` for custom output.

2. Module-level `console = Console()`

3. Click group `cli`:
   - `@click.group()` decorator
   - `@click.version_option(package_name="mdc-encyclopedia")` for --version
   - `@click.pass_context` to pass db_path via context
   - In the callback body:
     - `ctx.ensure_object(dict)`
     - Read db_path from `os.environ.get("MDC_ENCYCLOPEDIA_DB", "mdc_encyclopedia.db")`
     - Call `init_db(db_path)` from db module -- capture the boolean return
     - If init_db returned True (new DB): `console.print(f"[green]✓[/green] Created database: {db_path}")`
     - Store db_path in `ctx.obj["db_path"] = db_path`
   - Docstring: `"""MDC Open Data Encyclopedia - catalog, enrich, and publish Miami-Dade County open datasets."""`

4. Seven stub subcommands, each as `@cli.command()` with descriptive docstring and body printing `console.print("[yellow]Not yet implemented[/yellow]")`:
   - `pull` -- "Pull dataset metadata from Miami-Dade open data portals."
   - `enrich` -- "Enrich datasets with AI-generated descriptions and metadata."
   - `audit` -- "Run quality audit on all datasets."
   - `diff` -- "Show changes between the current and previous catalog pulls."
   - `export` -- "Generate static HTML site from the catalog database."
   - `serve` -- "Start a local preview server for the static site."
   - `stats` -- "Show database summary: dataset counts, enrichment status, quality overview."

5. Import at top: `import os`, `import rich_click as click`, `from rich.console import Console`, `from mdc_encyclopedia.db import init_db`
