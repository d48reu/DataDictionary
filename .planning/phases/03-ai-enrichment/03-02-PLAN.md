**Command logic:**

1. **Get unenriched datasets**: Call `get_unenriched_datasets(conn)`. Resume is implicit -- this always returns only datasets without enrichment records. The `--resume` flag exists for user clarity but the behavior is the same (always skip enriched). If no unenriched datasets found, print "[green]All datasets already enriched.[/green]" and return.

2. **Apply --limit**: If `--limit N` is provided, slice the unenriched list to first N datasets.

3. **Check API key early** (unless --dry-run): Try `create_enrichment_client(model=model)`. If it raises `ValueError` (missing key), print the error in red and abort. For --dry-run, skip this check since no API calls are made.

4. **Build column lookup**: Query `get_columns_for_dataset(conn, dataset["id"])` for each unenriched dataset. Build a dict: `columns_by_dataset = {ds["id"]: get_columns_for_dataset(conn, ds["id"]) for ds in unenriched}`.

5. **Cost estimation**: Call `estimate_cost(client_or_None, model, unenriched, columns_by_dataset)`.
   - For --dry-run: Create the client just for cost estimation (count_tokens needs a client). Display results in a Rich table: dataset count, estimated input tokens, estimated output tokens, estimated cost. Also display a table listing each unenriched dataset title (truncated to 60 chars). Print the cost table and return.
   - For normal run: Display the cost estimate.

6. **Confirmation flow** (normal run only):
   - If `cost["total_est"] > 5.0`: Print cost and use `click.confirm("Estimated cost exceeds $5. Proceed?", abort=True)`
   - If `cost["total_est"] <= 5.0`: Print cost and auto-proceed with a message like "[green]Estimated cost: ${cost:.2f} (under $5 threshold, auto-proceeding)[/green]"

7. **Enrichment loop with Rich progress**:
   - Use a Rich Progress bar similar to the pull command (SpinnerColumn, TextColumn, BarColumn, TaskProgressColumn, TimeElapsedColumn)
   - For each dataset in unenriched list:
     a. Call `enrich_dataset(client, model, dataset, columns_by_dataset.get(dataset["id"], []))`
     b. Call `insert_enrichment(conn, dataset["id"], result.model_dump(), PROMPT_VERSION)` -- use `result.model_dump()` to convert Pydantic model to dict
     c. Advance progress bar
     d. Add a 1-second delay between API calls (time.sleep(1)) for rate limiting consistency with project patterns
   - Wrap the API call in try/except to handle individual dataset failures gracefully: log warning, increment failed_count, continue to next dataset (same graceful degradation pattern as pull command)

8. **Summary table**: After loop completes, display a Rich Table with:
   - Total enriched count
   - Failed count
   - Model used
   - Prompt version
   - Elapsed time

9. **Cleanup**: Close connection.

**Important implementation details:**
- The `--resume` flag does not change behavior -- it exists for UX clarity. The query always returns unenriched datasets. Document this in the flag help text: "Resume enrichment from where it left off (skips already-enriched datasets)."
- Use `time.sleep(1)` between API calls, matching the project's rate limiting pattern from hub_client.py
