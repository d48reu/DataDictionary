**src/mdc_encyclopedia/ingestion/field_fetcher.py:** Create the ArcGIS REST field metadata fetcher.

Imports: `httpx`, `time`, `tenacity` (retry, stop_after_attempt, wait_exponential), `normalizer.normalize_field` from the sibling module.

Constants:
- `RATE_LIMIT_SECONDS = 1.0`

Functions:

1. `fetch_service_info(client: httpx.Client, service_url: str) -> dict` -- Decorated with `@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=2, min=2, max=8), reraise=True)`. Calls `time.sleep(RATE_LIMIT_SECONDS)` before the request. GETs `{service_url}?f=json` with the httpx client. Calls `raise_for_status()`. Returns parsed JSON. Handles the case where the response is HTML (ArcGIS sometimes returns an HTML error page) by checking content-type or trying json parse and raising an appropriate error.

2. `fetch_layer_fields(client: httpx.Client, layer_url: str) -> list[dict]` -- Decorated with `@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=2, min=2, max=8), reraise=True)`. Calls `time.sleep(RATE_LIMIT_SECONDS)` before the request. GETs `{layer_url}?f=json`. Returns the `fields` list from the response, or empty list if no fields key.

3. `fetch_fields_for_dataset(client: httpx.Client, dataset_id: str, service_url: str | None) -> list[dict]` -- Main entry point. If `service_url` is None or empty, return empty list (File Geodatabase case -- per Pitfall 4 from research). Otherwise:
   - Call `fetch_service_info(client, service_url)` to get service metadata
   - Check if `fields` key exists directly (URL points to a specific layer) -- if so, normalize and return those fields
   - Check `layers` list -- for each layer, call `fetch_layer_fields(client, f"{service_url}/{layer['id']}")` and normalize each field with the layer name
   - Return all normalized fields as a list of dicts ready for `upsert_columns`
   - Wrap the whole function body in try/except: on any exception after retries exhausted, log a warning and return empty list (don't crash the whole pull for one dataset's fields)

Use `from mdc_encyclopedia.ingestion.normalizer import normalize_field` for field normalization.

Research confirmed that no Socrata portal exists for Miami-Dade County (Discovery API returns "Domain not found"), so the two-stage design collapses to a single "ArcGIS Hub" stage. The progress bar and summary table format are otherwise faithful to the locked decision.

1. **Stage 1 - Catalog Fetch:**
   - Creates httpx client via `create_client()`
   - Gets db connection via `get_connection(ctx.obj["db_path"])`
   - Initializes counters: `new_count`, `updated_count`, `skipped_count`, `failed_datasets` (list of (id, error))
   - Uses Rich Progress with columns: SpinnerColumn, TextColumn for description, BarColumn, TaskProgressColumn, TextColumn("datasets"), TimeElapsedColumn
   - Creates progress task "ArcGIS Hub" with total from first page's `numberMatched`
   - Iterates `fetch_all_datasets(client)`, for each `(feature, index, total)`:
     - On first iteration, update progress task total to `total` (actual `numberMatched`)
     - Normalize via `normalize_hub_dataset(feature)`
     - Upsert via `upsert_dataset(conn, normalized)`, count new vs updated
     - Advance progress bar by 1
     - Wrap in try/except: on error, increment skipped, append to failed_datasets, continue

2. **Stage 2 - Field Metadata Fetch:**
   - Query all datasets from DB: `SELECT id, api_endpoint FROM datasets WHERE api_endpoint IS NOT NULL`
   - Create new Rich Progress task "Fetching fields" with total = count of datasets with endpoints
   - For each dataset:
     - Call `fetch_fields_for_dataset(client, dataset_id, api_endpoint)`
     - Call `upsert_columns(conn, dataset_id, fields)` if fields returned
     - Advance progress bar
     - On error: log warning, continue

3. **Dedup Detection:**
   - Query all datasets: `SELECT id, title FROM datasets`
   - Build list of dicts and call `detect_duplicate_titles(datasets)` (uses normalized title matching per locked decision)
   - If duplicates found, print a Rich warning panel listing them
   - Store the duplicate count for the summary table

4. **Summary Table:**
   - Commit the database connection
   - Build a `Rich Table` with title "Pull Summary" containing:
     - Portal row: "ArcGIS Hub: N datasets" (per-portal count per locked decision; only one portal exists since research confirmed no Socrata portal)
     - Total datasets (new + updated)
     - New datasets count
     - Updated datasets count
     - Deduplicated count: number of within-catalog title duplicates detected and reported (per locked decision: summary table must include deduplicated count)
     - Datasets with fields (query `SELECT COUNT(DISTINCT dataset_id) FROM columns`)
     - Skipped/failed count
     - Elapsed time (formatted as Xm Ys)
   - Print the table via `console.print(table)`
   - If failed_datasets is non-empty, print them in a separate table

5. **Cleanup:**
   - Close the database connection
   - Close the httpx client

Add `--verbose` flag to the pull command (per locked decision: dedup reporting shows summary by default, detailed per-pair list with `--verbose`). When verbose, print each duplicate pair.

Important: The pull command does NOT need `@click.pass_context` on the function signature if using `click.get_current_context()` inside -- but the cleaner pattern is `@click.pass_context` with `ctx` as the first parameter.
