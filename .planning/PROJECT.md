# Miami-Dade County Open Data Encyclopedia

## What This Is

A CLI tool and static encyclopedia that catalogs, AI-enriches, and serves every dataset from Miami-Dade County's ArcGIS Open Data Hub as a searchable website. Residents, journalists, and civic hackers can browse, search, and understand what data the county publishes — with plain-English descriptions, quality scores, and practical use cases generated by Claude. Deployed to GitHub Pages with weekly auto-refresh.

## Core Value

Every Miami-Dade open dataset is discoverable and understandable by a non-technical resident.

## Requirements

### Validated

- ✓ Pull full catalog metadata from ArcGIS Hub API — v1.0
- ✓ Store all metadata in local SQLite database (datasets, columns, enrichments, audits) — v1.0
- ✓ AI-enrich each dataset with plain-English descriptions, use cases, owning department, update frequency, search keywords, and civic relevance — v1.0
- ✓ Run staleness/quality audit flagging stale, empty, or poorly described datasets — v1.0
- ✓ Detect changes between pulls (new, removed, schema changes) and surface on site — v1.0
- ✓ Generate static HTML site with search (Lunr.js), category browsing, dataset pages with quality scores and related datasets, and a data quality report — v1.0
- ✓ Deploy to GitHub Pages with weekly auto-refresh via GitHub Actions — v1.0
- ✓ CLI tool installable via pip as `mdc-encyclopedia` with commands: pull, stats, enrich, audit, diff, export, serve — v1.0

### Active

### Out of Scope

- Data preview (showing actual rows) — extra API cost, source portal already provides this
- Real-time data — this is a catalog, not a data warehouse
- OAuth or user accounts — pure static site, no auth needed
- Mobile app — responsive static site works well on mobile
- Email/Slack notifications — requires notification infrastructure and user accounts
- Hosted API for programmatic access — conflicts with static site architecture
- Comments/discussion on datasets — requires moderation and backend
- Embeddable dataset card widgets — v2 consideration

## Context

**Shipped v1.0** with 5,573 LOC across Python (3,416), HTML (705), CSS (1,150), and JS (302).

**Tech stack:** Python 3.12+, SQLite, Click/rich-click CLI, Jinja2 templates, Anthropic Python SDK, Lunr.js client-side search, vanilla HTML/CSS/JS, GitHub Actions, GitHub Pages.

**Data source:** ArcGIS Open Data Hub at gis-mdc.opendata.arcgis.com (Hub API v3). Research confirmed Miami-Dade has no active Socrata presence — all catalog data comes from ArcGIS Hub.

**Dataset count:** 570+ datasets discovered on first pull from ArcGIS Hub.

**Known tech debt:**
- browse.html grade CSS class mismatch (cosmetic, one-line fix)
- Push-trigger deploys empty site before first cron run (self-resolving)
- 15 human verification items pending (live API testing, terminal rendering, browser testing, GitHub infra)

## Constraints

- **Rate limiting**: ArcGIS Hub rate-limited to 1 request/second
- **Static site**: Must work without a server — pure static files, no backend at runtime
- **Cost transparency**: AI enrichment must show cost estimate and support dry-run before spending API credits
- **Packaging**: Installable via `pip install .` with `pyproject.toml` entry point for `mdc-encyclopedia` CLI
- **Tech stack**: Python 3.12+, SQLite, Click CLI, Jinja2 templates, Anthropic Python SDK, Rich terminal output, vanilla HTML/CSS/JS

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| SQLite over Postgres | Single-file DB, no server needed, works in CI and locally | ✓ Good — DB committed to repo for CI persistence |
| Lunr.js for search | Client-side search avoids backend, works with static hosting | ✓ Good — field boosts (title 10x, keywords 8x) give relevant results |
| GitHub Pages deployment | Free hosting, integrates with Actions for weekly refresh | ✓ Good — artifact-based deploy, no gh-pages branch needed |
| No data preview | Avoids extra API calls per dataset, keeps scope focused on discovery | ✓ Good — source portal links serve this need |
| Enrich new datasets only (weekly) | Controls API cost while keeping new datasets enriched automatically | ✓ Good — 25-dataset threshold triggers enrichment in CI |
| Both local + CI enrichment | Flexibility — develop locally, optionally enrich in CI | ✓ Good — ANTHROPIC_API_KEY optional in Actions |
| ArcGIS Hub only (no Socrata) | Research confirmed Miami-Dade has no active Socrata portal | ✓ Good — simplified client code, one API to maintain |
| PRAGMA user_version for schema | Lightweight versioning without extra tables | ✓ Good — V1→V2 migration worked cleanly |
| Synchronous httpx (not async) | 1 req/s rate limit makes async pointless | ✓ Good — simpler code, same performance |
| XDG base directory for DB | Standard location, no platformdirs dependency | ✓ Good — ~/.local/share/mdc-encyclopedia/ |
| python-slugify for URLs | Collision detection via dataset ID suffix | ✓ Good — clean URLs at /dataset/{slug}/ |
| Artifact-based Pages deploy | No gh-pages branch needed, cleaner workflow | ✓ Good — standard GitHub approach |
| DB committed to repo | actions/cache evicts after 7 days, too risky for data persistence | ✓ Good — [skip ci] prevents infinite loops |
