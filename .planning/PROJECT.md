# Miami-Dade County Open Data Encyclopedia

## What This Is

A public-facing tool that catalogs, AI-enriches, and serves every dataset from Miami-Dade County's two open data portals (Socrata and ArcGIS Open Data Hub) as a searchable static encyclopedia. Residents, journalists, and civic hackers can browse, search, and understand what data the county publishes — with plain-English descriptions, quality scores, and practical use cases generated by Claude.

## Core Value

Every Miami-Dade open dataset is discoverable and understandable by a non-technical resident.

### Out of Scope

- Data preview (showing actual rows) — extra API cost per dataset, not needed for discovery
- Real-time data — this is a catalog, not a data warehouse
- OAuth or user accounts — pure static site, no auth needed
- Mobile app — responsive static site is sufficient
- Notifications (email/Slack) for changes — v2 consideration

## Context

**Data sources:**
1. **Socrata portal:** opendata.miamidade.gov — Discovery API at `https://api.us.socrata.com/api/catalog/v1?domains=opendata.miamidade.gov` (paginates at 100 results). Column metadata requires per-dataset call to `/api/views/{id}.json`.
2. **ArcGIS Open Data Hub:** gis-mdc.opendata.arcgis.com — Hub API at `/api/v3/datasets`.

**Dataset count:** Unknown until first pull — could be hundreds to low thousands across both portals.

**Audience:** Public tool for residents, journalists, and civic hackers exploring Miami-Dade County data. Not an official county product.

**Enrichment approach:** Claude API generates plain-English descriptions, practical use cases, likely owning department, expected update frequency, search keywords a resident would use, and why a resident or journalist would care. Supports `--dry-run` (show what would be enriched), `--resume` (pick up where left off), and cost estimate before running.

**Weekly refresh behavior:** GitHub Action pulls fresh metadata weekly. Newly discovered datasets are auto-enriched. Already-enriched datasets keep their enrichment unless manually re-run.

**API key handling:** ANTHROPIC_API_KEY works both locally and as a GitHub Secret. Enrichment is optional in CI — Actions can run pull + export without enrichment if key isn't set.

**Static site features:**
- Client-side search via Lunr.js or Fuse.js (JSON index built at export time)
- Category browsing pages
- Individual dataset pages showing: metadata, AI enrichment, column info, quality score, related datasets, source portal link
- "What Changed" page showing diff results between pulls
- Data quality report page
- Footer on every page: "Not an official Miami-Dade County product. Built by Abreu Data Works LLC."

**Branding:** MDC blue (#003366), white, amber for warnings/staleness indicators.

## Constraints

- **Rate limiting**: Both portals rate-limited to 1 request/second
- **Socrata pagination**: Discovery API returns max 100 results per request — must paginate
- **Column metadata**: Requires separate API call per dataset on Socrata (`/api/views/{id}.json`)
- **Static site**: Must work without a server — pure static files, no backend at runtime
- **Cost transparency**: AI enrichment must show cost estimate and support dry-run before spending API credits
- **Packaging**: Installable via `pip install .` with `pyproject.toml` entry point for `mdc-encyclopedia` CLI
- **Tech stack**: Python 3.12+, SQLite via sqlite-utils, Click CLI, Jinja2 templates, Anthropic Python SDK, Rich terminal output, vanilla HTML/CSS/JS
