Add four new functions to the bottom of `src/mdc_encyclopedia/db.py`:

1. `get_dataset_ids(conn) -> set[str]`: Query `SELECT id FROM datasets`, return a set of all dataset ID strings. Used for pre/post snapshot comparison.

2. `get_columns_snapshot(conn) -> dict[str, set[str]]`: Query `SELECT dataset_id, name FROM columns`, return a dict mapping each dataset_id to a set of its column name strings. Datasets with no columns will not appear in the dict (empty set is implicit -- treat missing key as empty set during comparison).

3. `insert_change(conn, dataset_id, change_type, details=None, detected_at=None)`: Insert a row into the `changes` table. If `detected_at` is provided (a string), use it instead of the DEFAULT so all changes from one pull share exactly the same timestamp. SQL: `INSERT INTO changes (dataset_id, change_type, details, detected_at) VALUES (?, ?, ?, ?)` when detected_at is given, else `INSERT INTO changes (dataset_id, change_type, details) VALUES (?, ?, ?)` to use the DEFAULT. Do NOT commit inside this function -- the caller batches inserts and commits once.

4. `get_recent_changes(conn, limit=None) -> list[dict]`: Query changes joined with datasets for titles. Use LEFT JOIN from changes to datasets (removed datasets may still have a row, but LEFT JOIN is defensive). Order by `detected_at DESC, change_type, dataset_id`. Apply LIMIT if provided. Return list of dicts with keys: id, dataset_id, change_type, details, detected_at, title.

Follow existing db.py conventions: type hints, docstrings matching the existing style, sqlite3.Row-based results converted to dicts.

<task type="auto">
  <name>Task 2: Create diff/detector.py with snapshot capture and change computation</name>

  <action>
Create the `src/mdc_encyclopedia/diff/` package:

1. `__init__.py`: Empty file or single-line docstring `"""Change detection module."""`

2. `detector.py`: Contains two public functions:

**`capture_snapshot(conn) -> dict`**
- Calls `get_dataset_ids(conn)` and `get_columns_snapshot(conn)` from db.py.
- Returns `{"dataset_ids": set[str], "columns_by_dataset": dict[str, set[str]]}`.
- This is a thin wrapper that bundles the two queries into a single snapshot dict.

**`compute_changes(conn, pre_snapshot, post_snapshot) -> dict`**
- Accepts the connection and pre/post snapshots from `capture_snapshot`.
- Computes three categories using set operations:
  - `added_ids = post_ids - pre_ids`
  - `removed_ids = pre_ids - post_ids`
  - `schema_changes`: For each dataset in `pre_ids & post_ids`, compare column name sets. If different, record `(dataset_id, columns_added, columns_removed)`.
- Generates a shared `detected_at` timestamp using `datetime.now(timezone.utc).strftime("%Y-%m-%d %H:%M:%S")` ONCE at the start, passed to every `insert_change` call. This guarantees all changes from one pull share exactly the same timestamp (addresses the research open question about SQLite datetime resolution).
- For "added" changes: call `insert_change(conn, ds_id, "added", None, detected_at)`.
- For "removed" changes: Look up the dataset title from the datasets table (it should still be there since pull never deletes rows). Store `json.dumps({"title": title})` in the details field as a defensive measure per research recommendation. Call `insert_change(conn, ds_id, "removed", details, detected_at)`.
- For "schema_changed" changes: Store `json.dumps({"columns_added": sorted(list), "columns_removed": sorted(list)})` in details. Call `insert_change(conn, ds_id, "schema_changed", details, detected_at)`.
- After all inserts, call `conn.commit()` once.
- Return `{"added": len(added_ids), "removed": len(removed_ids), "schema_changed": len(schema_changes)}`.

**Edge cases:**
- If pre_snapshot dataset_ids is empty (first pull), return `{"added": 0, "removed": 0, "schema_changed": 0}` immediately without inserting anything. The caller (pull command) handles the first-pull message.
- If pre and post are identical, all set differences are empty, no records inserted, return zeros.
- Treat missing key in columns_by_dataset as empty set (use `.get(ds_id, set())`).

Import from mdc_encyclopedia.db: get_dataset_ids, get_columns_snapshot, insert_change.
