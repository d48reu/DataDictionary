**site/search_index.py**: Search index builder module.
- `build_search_index(datasets: list[dict], output_dir: str) -> dict` — builds Lunr.js-compatible search index using lunr.py.
  - Create documents list: each doc has id (slug), title, description (ai_description or description, first 200 chars), keywords (joined keywords list), department, category.
  - Create search_data dict keyed by slug: title, snippet (ai_description[:120] or description[:120]), department, url (/dataset/{slug}/), grade.
  - Build index with `lunr()`: ref="id", fields with boost: title (10), keywords (8), description (5), department (3), category (3).
  - Write search-index.json (serialized index) and search-data.json to output_dir.
  - Return dict with index_size (bytes of search-index.json) and data_size (bytes of search-data.json).
  - Handle empty datasets list gracefully (write empty index).

**site/static/search.js**: Full client-side search logic (replace stub from Plan 01).
- On DOMContentLoaded, call `initSearch()` which fetches /search-index.json and /search-data.json via Promise.all. Store in module-level variables `searchIndex` and `searchData`.
- `onSearchInput(query)` — triggered on keyup from search input. If query < 2 chars, hide dropdown. Otherwise, run `searchIndex.search(query + '*')` for as-you-type matching. Take top 8 results. Map results to searchData entries. Call `renderDropdown(items)`.
- `renderDropdown(items)` — creates/updates a `.search-dropdown` div positioned below the search input. Each result item is a clickable link showing: dataset title (bold), 1-line snippet (muted), department chip. If no results, show "No datasets found" message.
- `hideDropdown()` — removes the dropdown div.
- Click-outside handler: hide dropdown when clicking outside the search container.
- Handle index loading state: disable search input until index is loaded, show "Loading search..." placeholder.
- Lunr.js is loaded from /static/lunr.min.js (vendored) or CDN. Use CDN reference in base.html: `<script src="https://cdn.jsdelivr.net/npm/lunr@2.3.9/lunr.min.js"></script>`.

**site/generator.py**: Update to integrate search index building.
- Import `build_search_index` from `search_index` module.
- In `generate_site()`, after rendering all pages, call `build_search_index(site_data["datasets"], output_dir)`.
- Include index stats in return dict (index_size_kb).
- Add Lunr.js CDN script tag to base.html head block or ensure it is already there from Plan 01.
