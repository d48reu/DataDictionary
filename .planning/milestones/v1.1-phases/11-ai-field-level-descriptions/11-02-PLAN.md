  <action>
  Add a new `enrich_fields_cmd` Click command to the CLI group. Follow the exact pattern of the existing `enrich` command (around lines 314-465 in cli.py) for structure, imports, and Rich output.

  **Command signature:**
  ```python
  @cli.command("enrich-fields")
  @click.option("--db", default=None, help="Database path")
  @click.option("--dry-run", is_flag=True, help="Show eligible datasets and cost estimate without enriching")
  @click.option("--limit", type=int, default=None, help="Max datasets to enrich")
  @click.option("--resume", is_flag=True, help="Skip already-enriched datasets")
  @click.option("--model", default="claude-haiku-4-5-20241022", help="Claude model")
  ```

  **Implementation flow:**

  1. **Setup:** Get DB connection (same pattern as enrich command). Validate ANTHROPIC_API_KEY from environment (fail fast if missing, unless --dry-run).

  2. **Query eligible datasets:** Call `get_field_eligible_datasets(conn)`. This already filters by B+ grade and unenriched columns.

  3. **Resume filtering:** If `--resume`, the query already returns only datasets with unenriched columns. Track the count of fully-enriched datasets separately for the end summary by querying B+ datasets WHERE all columns have ai_description. Show "Skipped N already-enriched datasets" in summary.

  4. **Limit application:** If `--limit`, slice the eligible list to first N.

  5. **Dry-run mode:** If `--dry-run`:
     - Count total eligible datasets and total columns (sum of total_columns from query results)
     - Estimate cost using `estimate_cost()` pattern from existing enrich command (or approximate: ~800 input tokens per dataset avg, ~400 output tokens per dataset avg, compute with model pricing)
     - Display summary line: "Found {N} eligible datasets with {M} columns (~${cost:.2f} estimated)"
     - Display Rich Table with columns: Dataset, Jurisdiction, Grade, Columns, with one row per eligible dataset
     - Exit without API calls

  6. **Enrichment loop:** For each eligible dataset:
     - Print progress: `console.print(f"Enriching {dataset['title']} ({dataset['total_columns']} columns)...", end="")`
     - Get full column data: call `get_columns_for_dataset(conn, dataset['id'])`
     - Call `enrich_fields(client, model, dataset, columns)` from client.py
     - On success: call `update_column_ai_descriptions(conn, dataset['id'], {fd.column_name: fd.description for fd in result.field_descriptions})`
     - Print completion: `" done [{current}/{total}]"`
     - On API error: retry once after 2-second delay. If still fails, log warning, increment failed counter, continue to next dataset. Do NOT crash.
     - Add 1-second delay between datasets (matching existing rate limiting pattern)

  7. **End summary:** Display Rich Panel with:
     - Datasets enriched: N
     - Columns described: M (sum of field_descriptions returned)
     - Skipped (already enriched): K (if --resume)
     - Failed: F (if any)
     - List of failed dataset titles (if any)

  **Imports needed:** Import `get_field_eligible_datasets`, `get_columns_for_dataset`, `update_column_ai_descriptions` from db. Import `enrich_fields` from enrichment.client. Import `FieldEnrichmentResult` from enrichment.models (for type hint only). Use existing `anthropic`, `time`, `Console`, `Table`, `Panel` imports that are already in cli.py.
