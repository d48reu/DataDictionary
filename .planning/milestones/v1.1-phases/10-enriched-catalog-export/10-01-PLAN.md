Create `src/mdc_encyclopedia/site/catalog_export.py` with two public functions:

**`generate_catalog_json(site_data, output_dir) -> dict`:**
- Build root catalog object: `{"conformsTo": "https://project-open-data.cio.gov/v1.1/schema", "@type": "dcat:Catalog", "dataset": [...]}`
- Map each dataset from `site_data["datasets"]` to DCAT-US v1.1 using a private `_dataset_to_dcat(ds)` function
- Required DCAT fields to map: `@type` (constant "dcat:Dataset"), `title`, `description` (prefer ai_description, fallback to description), `keyword` (use keywords_list or tags_list), `modified` (updated_at), `publisher` (nested org:Organization object), `identifier` (ds["id"])
- Optional DCAT fields (include only when value exists, NO nulls): `license`, `spatial` (bbox), `issued` (created_at), `landingPage` (source_url), `theme` (category wrapped in array), `accrualPeriodicity` (update_freq)
- Distribution array: build from download_url (mediaType: text/csv, format: CSV) and api_endpoint (format: API). Only add distribution key if at least one exists
- Extension field: `jurisdiction` (ds["jurisdiction"]) -- non-DCAT, our extension
- AI enrichment extensions with `ext_` prefix (include only when value exists): `ext_aiDescription`, `ext_qualityGrade` (letter_grade), `ext_qualityScore` (composite_score), `ext_civicRelevance`, `ext_department`, `ext_useCases` (use_cases_list)
- Write to `catalog.json` in output_dir using `json.dump()` with `indent=2, ensure_ascii=False`
- Compute file size AFTER the with-block closes using `os.path.getsize()`
- Return `{"dataset_count": len(datasets), "file_size": file_size_bytes}`

**`generate_catalog_csv(site_data, output_dir) -> dict`:**
- Define explicit CSV_COLUMNS list for column ordering (human-friendly headers per Claude's discretion): ID, Title, Description, Jurisdiction, Category, Publisher, Keywords, Last Modified, Created, Quality Grade, Quality Score, AI Description, Civic Relevance, Department, Use Cases, Format, Source URL, Download URL, API Endpoint, License
- Create private `_flatten_for_csv(ds)` function mapping dataset dict to flat row dict with semicolon-joined multi-value fields (keywords, use cases)
- Open output file with `encoding='utf-8-sig'` (writes BOM bytes automatically) and `newline=''`
- Use `csv.DictWriter` with `fieldnames=CSV_COLUMNS, extrasaction='ignore'`
- Write header row then iterate `site_data["datasets"]` writing flattened rows
- Compute file size AFTER the with-block closes
- Return `{"dataset_count": len(datasets), "file_size": file_size_bytes}`
