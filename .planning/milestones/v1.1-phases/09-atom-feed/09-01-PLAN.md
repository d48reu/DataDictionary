    **1. Add feedgen dependency to pyproject.toml:**
    Add `"feedgen>=1.0,<2.0",` to the `dependencies` list in `[project]`.

    **2. Install the updated package:**
    Run `pip install -e .` to install feedgen and its dependencies (lxml, python-dateutil).

    **3. Update context.py changes query to include jurisdiction:**
    In `build_site_data()`, the changes query (around line 117-124) currently does:
    ```sql
    SELECT c.*, d.title AS dataset_title
    FROM changes c
    LEFT JOIN datasets d ON c.dataset_id = d.id
    ORDER BY c.detected_at DESC
    ```
    Add `d.jurisdiction` to the SELECT:
    ```sql
    SELECT c.*, d.title AS dataset_title, d.jurisdiction
    FROM changes c
    LEFT JOIN datasets d ON c.dataset_id = d.id
    ORDER BY c.detected_at DESC
    ```
    This makes jurisdiction available in each change record for feed summaries. For removed datasets (where `d.jurisdiction` may be NULL due to LEFT JOIN), the feed summary will omit jurisdiction gracefully.

    **4. Create `src/mdc_encyclopedia/site/feed.py`:**
    New module with a single public function `generate_atom_feed(site_data, output_dir, site_url, max_entries=50)`.

    Implementation details:
    - Import `FeedGenerator` from `feedgen.feed`
    - Import `os`, `datetime`/`timezone` from stdlib
    - Strip trailing slash from `site_url`
    - Configure feed metadata:
      - `fg.id(f"{site_url}/atom.xml")` — permanent unique URI per RFC 4287
      - `fg.title("MDC Data Encyclopedia - Catalog Changes")`
      - `fg.subtitle("New, removed, and changed open datasets across South Florida")`
      - `fg.author({"name": "MDC Data Encyclopedia"})`
      - `fg.link(href=site_url, rel="alternate")` — link to the site itself
      - `fg.link(href=f"{site_url}/atom.xml", rel="self")` — canonical feed URL
      - `fg.language("en")`
    - Build a slug lookup dict: `{ds["id"]: ds["slug"] for ds in site_data["datasets"]}`
    - Define PREFIX_MAP: `{"added": "[New]", "removed": "[Removed]", "schema_changed": "[Schema Change]"}`
    - Slice `site_data["changes"][:max_entries]` for the most recent changes
    - Set feed `updated` to the most recent change's `detected_at` timestamp, or `datetime.now(timezone.utc)` if no changes exist
    - For each change record, create a feed entry:
      - `fe.id()` — use tag URI format: `tag:{domain},{date}:{dataset_id}/{change_type}` where domain is extracted from site_url (strip protocol). Example: `tag:example.github.io/DataDictionary,2026-02-27:miami-dade_abc123/added`
      - `fe.title()` — `f"{prefix} {title}"` using PREFIX_MAP (per user decision: prefix tag format)
      - `fe.link()` — absolute URL `f"{site_url}/dataset/{slug}/"` using slug_lookup. If slug not found (removed dataset), omit the link.
      - `fe.summary()` — call `_build_summary(change, change_type)` (see below)
      - `fe.updated()` and `fe.published()` — parse `detected_at` string to timezone-aware datetime
    - Write feed: `fg.atom_file(os.path.join(output_dir, "atom.xml"), pretty=True)`
    - Return `{"entry_count": len(changes)}`

    Helper `_parse_timestamp(ts_str)`:
    - `datetime.fromisoformat(ts_str)`, if tzinfo is None attach `timezone.utc`
    - On any error, return `datetime.now(timezone.utc)`

    Helper `_build_summary(change, change_type)`:
    - Extract title from `change.get("dataset_title") or change.get("title") or "a dataset"`
    - Extract jurisdiction from `change.get("jurisdiction", "")`
    - Per user decision: 1-2 sentence summary mentioning jurisdiction when available:
      - added + jurisdiction: `f"A new dataset '{title}' has been added from {jurisdiction}."`
      - added no jurisdiction: `f"A new dataset '{title}' has been added to the catalog."`
      - removed + jurisdiction: `f"The dataset '{title}' from {jurisdiction} has been removed from the catalog."`
      - removed no jurisdiction: `f"The dataset '{title}' has been removed from the catalog."`
      - schema_changed + jurisdiction: `f"The schema for '{title}' from {jurisdiction} has changed. Columns may have been added or removed."`
      - schema_changed no jurisdiction: `f"The schema for '{title}' has changed. Columns may have been added or removed."`
      - default: `f"A change was detected for '{title}'."`
    - Use the jurisdiction's display name, not slug. The `jurisdiction` field from the DB is the slug (e.g., "miami-dade"). To get human-readable form, title-case the slug and replace hyphens with spaces: `jurisdiction.replace("-", " ").title()`. This gives "Miami Dade" which is close enough — exact display_name lookup would require loading the registry, which is overkill for a summary.

    **Key pitfall avoidances (from RESEARCH.md):**
    - All datetimes timezone-aware (Pitfall 4)
    - All URLs absolute using site_url (Pitfall 1)
    - Entry IDs use tag URI with dataset_id + change_type + date for stability across rebuilds (Pitfall 3)
    - Empty feed is valid — generate even with zero entries (Pitfall 5)
