### Missing Connection (1)

**`enrich` CLI command does not pass jurisdiction to `enrich_dataset()`**

- `get_unenriched_datasets()` in `db.py` does not SELECT `d.jurisdiction`
- `cli.py:enrich()` calls `enrich_dataset()` without `jurisdiction=` kwarg
- `build_system_prompt()` defaults to "Miami-Dade County" for all datasets
- Broward/Miami datasets get Miami-Dade-specific department lists in AI summaries
- The `enrich-fields` path is NOT affected (correctly wired)
- **Fix:** Add `d.jurisdiction` to SELECT, load registry in enrich command, pass display_name

## Tech Debt (3 items)

### HIGH: Jurisdiction not passed in `enrich` command

- **Files:** `db.py` (`get_unenriched_datasets`), `cli.py` (`enrich` command)
- **Impact:** Non-Miami-Dade datasets receive Miami-Dade-specific AI summaries
- **Not a blocker:** Infrastructure exists (`build_system_prompt()` works when called correctly). The `enrich-fields` path IS correct. Miami-Dade is the primary jurisdiction with 476 datasets; Broward/Miami have not been enriched yet.

### MEDIUM: `upsert_columns` wipes `ai_description` on re-pull

- **File:** `db.py` (`upsert_columns`, line 414)
- **Impact:** Standalone `pull` destroys field-level AI descriptions
- **Not a blocker:** CI pipeline runs `enrich-fields --resume` after `pull`, re-enriching any wiped fields. The `--resume` flag makes this idempotent.
