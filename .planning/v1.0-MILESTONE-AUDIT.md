## Tech Debt by Phase

### Phase 8: Multi-Jurisdiction Foundation (3 items)

1. **Dead helper function** — `get_datasets_by_jurisdiction()` in `db.py` line 675 has no callers. The `stats` command achieves the same result with inline SQL. Can be removed or wired into stats to deduplicate.

2. **Incomplete denormalization** — `datasets.ai_description` column is populated during V3 migration but not updated by `insert_enrichment()`. All downstream consumers (`context.py`, `catalog_export.py`, `feed.py`) read through the enrichments JOIN alias, so runtime behavior is correct. A future `UPDATE datasets SET ai_description = ? WHERE id = ?` in `insert_enrichment()` would close this gap.

3. **Deferred prompt integration** — `build_system_prompt(jurisdiction_display_name)` in `enrichment/prompts.py` is never called by `enrich_dataset()` in `client.py`. The enrichment client hardcodes `SYSTEM_PROMPT` (Miami-Dade department list). This was **explicitly deferred** to Phase 11 scope per 08-02 SUMMARY and STATE.md blockers section.
