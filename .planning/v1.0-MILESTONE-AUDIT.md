### Integration Issues Found

**1. Changes page scope (SITE-07) — Medium**
`context.py` fetches ALL rows from `changes` table with no WHERE clause limiting to the most recent `detected_at` batch. Template says "most recent catalog pull" but data spans all pulls. CLI `diff` command correctly scopes with `--latest`.

**2. Grade threshold mismatch (AUDIT-04, SITE-08) — Medium**
Three sources disagree on grade boundaries:
- `scorer.py`: B >= 0.75, D >= 0.40 (authoritative — writes to DB)
- `context.py` avg_grade: B >= 80%, D >= 60% (wrong — site aggregate)
- `quality.html` methodology: B (80-89), D (60-69) (wrong — user-facing text)

Per-dataset grades on dataset/browse pages use DB values (correct). Only the quality report page aggregate and methodology text are affected.

**3. Dead filter strings (SITE-07) — Low**
`generator.py` line 369 filters for `"columns_added"` and `"columns_removed"` but `detector.py` only inserts `"schema_changed"`. These branches never match.
