### E2E Flows: 2/2 complete

1. **pip install . -> mdc-encyclopedia pull**: Package installs, CLI resolves, DB initializes, catalog fetches, fields fetch, dedup runs, summary displays
2. **mdc-encyclopedia enrich**: Queries unenriched datasets, builds prompts, estimates cost, confirms, enriches via messages.parse(), inserts with immediate commit, shows progress

### Integration Findings (non-breaking)

1. **--resume flag is a no-op** (ENRICH-08): Flag is accepted but never branched on. Behavior is correct (always skips enriched datasets via LEFT JOIN), but the flag provides no distinct behavior. Low severity.

2. **Pydantic not a direct dependency**: `enrichment/models.py` imports pydantic but it's only available as a transitive dependency via `anthropic>=0.80`. If anthropic ever drops pydantic, the package breaks on install. Low risk.

3. **Cost estimate may understate**: `ESTIMATED_OUTPUT_TOKENS=500` but `max_tokens=1024`. Output cost could be up to 2x higher than estimated. Acceptable — the $5 confirmation threshold provides a safety net.
